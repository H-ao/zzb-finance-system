# 窗口关闭自动保存功能

## 🎉 功能说明

程序现在会在关闭窗口时自动保存数据并优雅地关闭后台进程，无需手动按 Ctrl+C！

---

## ✨ 关闭流程

### **窗口模式**

当你点击窗口右上角的 ✕ 按钮时：

1. 系统自动检测窗口关闭事件
2. 触发数据保存回调
3. 保存所有未保存的数据到数据库
4. 关闭所有数据库连接
5. 显示友好的退出提示
6. 程序完全退出

**退出提示**:
```
😊 窗口正在关闭，正在保存数据...
✓ 已保存所有数据
✓ 已关闭数据库连接
✓ 感谢使用，再见！
```

---

### **浏览器模式**

浏览器模式下，关闭窗口后需要按 **Ctrl+C** 退出程序：

```
Starting server on http://127.0.0.1:5000...
按 Ctrl+C 停止服务
关闭窗口后请按 Ctrl+C 退出程序
```

按 Ctrl+C 后：
```
😊 接收到退出信号，正在关闭...
✓ 已保存所有数据
✓ 已关闭数据库连接
✓ 感谢使用，再见！
```

---

## 🔧 技术实现

### 窗口关闭事件处理

```python
def on_closing():
    """窗口关闭时的回调函数"""
    print("\n😊 窗口正在关闭，正在保存数据...")
    
    # 等待数据保存完成
    time.sleep(1)
    
    # 关闭所有数据库连接
    _close_all_connections()
    
    print("✓ 已保存所有数据")
    print("✓ 已关闭数据库连接")
    print("✓ 感谢使用，再见！\n")

# 注册关闭事件
window.events.closing += on_closing
```

### 数据保护机制

1. **WAL 模式**: SQLite WAL 模式确保数据立即写入
2. **checkpoint**: 每次保存后执行 WAL 检查点
3. **优雅关闭**: 等待 1 秒确保所有数据落盘
4. **连接管理**: 正确关闭所有数据库连接

## ⚠️ 注意事项

### ✅ 安全关闭

关闭窗口后，系统会：
- 自动保存所有数据
- 等待 1 秒确保保存完成
- 优雅地关闭数据库连接
- 完全退出进程

**不用担心数据丢失！**

---

### ❌ 不要强制关闭

以下情况可能导致数据丢失：

1. **强制杀进程**
   ```bash
   kill -9 <PID>  # ❌ 可能导致数据丢失
   ```

2. **直接断电**
   - 在保存过程中断电会损坏数据库

3. **强制关窗口时正在保存**
   - 如果显示 "正在保存数据..." 时强制关闭窗口

---

   - 可能导致数据库锁竞争
